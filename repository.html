Repository information

Skysong have created a number of RPMs' to improve the ability to install complex software, for this purpose we allow the RPMs' that we have created to be available in an open repository to allow potential clients to see where we could help improve their software deployment and distribution.This site is also the home of the skysong yum repository of Oracle RDBMS installation and configuration RPMs for CentOS/Redhat servers.

The code below should be placed into a file in /etc/yum.repos.d nominally this should be called skysong.repo to separate this from the default archives in your distribution.

[skysong]

name=skysong - Base

baseurl=http://www.skysong.co.uk/yum/6/os/i386/CentOS/

gpgcheck=1

gpgkey=file:///www.skysong.co.uk/yum/RPM-GPG-KEY-skysong

enabled=1

To install the repository key the following command will need to be run: -

rpm --import http://skysong.co.uk/yum/RPM-GPG-KEY-skysong

The key can be checked with the following command: -

rpm -q gpg-pubkey --qf '%{name}-%{version}-%{release} --> %{summary}\n'

The line below will appear:-

gpg-pubkey-861ce91f-5149b67c --> gpg(Paul Tuhill (skysong) )

The Skysong archive repository holds the Skysong Oracle RDBMS RPMs (skysong-dbms-1 and skysong-dbms-2) and the Skysong oracle installation and configuration RPM (skysong). These are supplied to allow a non DBA to install the Oracle RDBMS easily and quickly to any RPM enabled version of Linux, these are supplied without any implied license for the Oracle software which should be purchased from the Vendor directly before downloading these RPMs or using the repository.

In the present iteration the Skysong oracle installation and configuration RPM (skysong) installs Oracle RDBMS 11g enterprise edition and will require appropriate licenses for this edition of the software
